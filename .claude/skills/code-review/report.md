# Final Report Compilation

Synthesize findings into actionable report with prioritized recommendations.

## Capabilities

No LSP tools used - This phase synthesizes data from all previous phases.

## Report Structure

### Executive Summary

- Overall assessment (rating out of 5)
- Critical issues count
- High priority improvements count
- Code quality score
- Tool availability summary

### Critical Issues (Must Fix)

- Issue description
- Exact location (file:line)
- Impact assessment
- Solution approach with code examples
- Verification steps

### High Priority Improvements

- Improvement description
- Affected locations
- Rationale
- Implementation approach

### Medium Priority Enhancements

- Enhancement description
- Benefits
- Implementation effort estimate

### Strengths to Maintain

- Architectural patterns working well
- Code quality highlights
- Best practices observed

### Capability Limitations

- Unsupported capabilities that limited analysis
- Fallback strategies used
- Impact on review comprehensiveness

### Refactoring Safety Analysis

- Safe refactoring candidates
- High-risk operations to avoid
- LSP support quality for refactoring

- Specific files to modify
- Exact changes needed
- Code examples
- Testing requirements
